# S026: Coverage Gate 80% — 修正 JaCoCo 配置 + 補齊測試覆蓋率

> Spec: S026 | Size: XS (7) | Status: ✅ Done
> Date: 2026-04-22

---

## 1. Goal

**一句話：** 修正 S025 的 JaCoCo gate bug（`rule.includes` 在 BUNDLE 層級無效），將覆蓋率策略從「domain + service」改為「全部程式碼扣除純接線」，閾值提升至 80%，並補齊 `TaskService` / `ProjectService` 的單元測試使 gate 通過。

### 背景

`/verifying-quality` 審查 [Unreleased] 批次時發現：

1. **S025 配置 bug：** `violationRules.rule.includes` 在預設 `element = BUNDLE` 下比對的是 bundle 名稱（"grimo"），不是 class 名稱。Pattern `io.github.samzhu.grimo.*.domain.*` 永遠不匹配 "grimo"，gate 形同虛設（任何閾值都通過）。
2. **策略不完整：** 只看 `domain/` + `application/service/` 會遺漏含業務邏輯的 adapter 和 internal 套件（如 `JdbcSessionEventAdapter` 的 recursive CTE、`ChatController` 的 session 狀態管理）。
3. **冗餘腳本：** `verify-tests-pass.sh` 被 `verify-all.sh` V2 取代；`verify-spec-coverage.sh` 被 `/verifying-quality` Step 4 取代。

### 依賴

無。純基礎設施 + 測試補齊。

---

## 2. Approach

### 2.1 JaCoCo gate 修正：`classDirectories` 過濾

```
問題：rule.includes 在 element=BUNDLE 下比對 bundle 名稱，非 class 名稱
修正：改用 classDirectories.setFrom(...matching { exclude(...) })
```

**排除清單（無業務邏輯的純接線）：**

| Pattern | 理由 |
|---------|------|
| `**/*Config.class`, `**/*Config$*.class` | `@Bean` / `@ComponentScan` 接線 |
| `**/port/in/**`, `**/port/out/**` | 純 interface，0 行可執行碼 |
| `**/events/**` | 純 Spring event record |
| `**/package-info.class` | 模組宣告 |
| `org/springaicommunity/**` | 第三方程式碼，非本專案維護範圍 |
| `**/sandbox/internal/**` | Docker 依賴，無法 unit test（技術債，待 S008 處理） |

### 2.2 閾值：75% → 80%

納入 gate 的程式碼約 805 行。目前覆蓋 603 行（74.9%）。需再覆蓋 41 行達到 80%（644 行）。

### 2.3 測試補齊目標

| Class | 目前 | missed | 測試策略 |
|-------|------|--------|---------|
| `TaskService` | 8% (4/51) | 47 行 | 純 unit test：mock `TaskPort` + `ProjectPort`，覆蓋 create/update/delete/list 的分支邏輯 |
| `ProjectService` | 10% (3/29) | 26 行 | 純 unit test：mock `ProjectPort`，覆蓋 duplicate-name guard、patch semantics、not-found guard |

補完這兩個 class 即可覆蓋 73 行（超額 32 行 buffer）。

**不包含：** `SessionHistoryService`（12 行，大部分是委派）、`DefaultContainerizedAgentModelFactory`（10 行，需真實 SDK）— 留給後續 spec 順手處理。

### 2.4 刪除冗餘腳本

| 腳本 | 替代方案 |
|------|---------|
| `verify-tests-pass.sh` | `verify-all.sh` V2 (`./gradlew test`) |
| `verify-spec-coverage.sh` | `/verifying-quality` Step 4（可測試性閘門） |

### 2.5 CSV 報告

新增 `csv.required.set(true)`，產生 `build/reports/jacoco/test/jacocoTestReport.csv`，每行一個 class，方便腳本解析覆蓋率缺口。

### 2.6 不包含

| 項目 | 理由 |
|------|------|
| sandbox.internal 的 unit test | Docker 依賴，待 S008 一併處理 |
| 逐 class 閾值 | 單一 BUNDLE 80% 足夠 MVP |
| CI badge / PR comment | 未來 CI spec 負責 |

```
Given  build.gradle.kts 使用 classDirectories 過濾（非 rule.includes）

**AC-2: TaskService 單元測試覆蓋主要分支**

```
Given  TaskServiceTest 存在

Then   所有測試通過
And    覆蓋：create（含 project 存在驗證）、list（含 "none" sentinel）、
       update（patch semantics）、updateStatus（含 isTerminal 分支）、delete
```

**AC-3: ProjectService 單元測試覆蓋主要分支**

```
Given  ProjectServiceTest 存在
When   ./gradlew test --tests ProjectServiceTest
Then   所有測試通過
And    覆蓋：create（含 duplicate-name guard）、update（含 patch + rename guard）、
       delete（含 not-found guard）、findById、findAll
```

**AC-4: 冗餘腳本已刪除，qa-strategy.md 已同步**

```
Given  verify-tests-pass.sh 和 verify-spec-coverage.sh 已刪除
When   讀取 qa-strategy.md
Then   §2 覆蓋率描述為「全部程式碼扣除純接線 ≥ 80%」
And    §6 管道不再引用已刪腳本
And    §6.1 V4 描述為「行覆蓋率 ≥ 80%（全部程式碼扣除純接線 + 第三方）」
And    §10 僅列 verify-all.sh

**AC-5: CSV 報告產生**

```
Given  build.gradle.kts 啟用 csv report
When   ./gradlew test jacocoTestReport
Then   build/reports/jacoco/test/jacocoTestReport.csv 存在
And    每行包含 GROUP, PACKAGE, CLASS, LINE_MISSED, LINE_COVERED 等欄位
```
