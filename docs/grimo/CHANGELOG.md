### 新增

- **S005 — 容器化 AgentModel 適配器**（2026-04-18）。在 `cli` 模組建立 `ContainerizedAgentModelFactory`（`@NamedInterface("api")`），為每個容器動態生成 docker exec wrapper script，透過各 SDK 的 binary path 設定注入（Claude `claudePath`、Gemini `GeminiAgentOptions.setExecutablePath()`、Codex `CodexAgentOptions.builder().executablePath()`）。完整重用 agent-client 0.12.2 的 `AgentModel` / `StreamingAgentModel` / `AgentResponse` 型別——零自建 AgentModel、零自建輸出解析。`StubContainerizedAgentModelFactory` 提供測試用存根（AC-1 ✅）。`ContainerizedAgentModelIT` 驗證對 grimo-runtime 容器的真實呼叫與 CLI 缺失錯誤處理（AC-2、AC-3，編譯通過待 Docker 驗證）。**附帶修正：** `cli` 模組 `allowedDependencies` 從 `{}` 改為 `{ "core" }`（Modulith 2.0.5 行為：即使 OPEN 模組仍需顯式宣告）；architecture.md、development-standards.md §13 同步修正此 drift。**里程碑 M2 完成。** 詳見 `docs/grimo/specs/archive/2026-04-18-S005-containerized-agent-cli-adapter.md` 第 7 節。
- **S004 — `grimo-runtime` Docker 映像**（2026-04-17）。建立 `docker/runtime/Dockerfile`，基於 `node:20-slim`（Debian/glibc），預裝三個 CLI 代理工具：Claude Code v2.1.112（curl 原生安裝器 — npm 套件已 deprecated）、Codex CLI 0.121.0（`@openai/codex`）、Gemini CLI 0.38.1（`@google/gemini-cli`）以及 git。映像大小 957 MB（< 1 GB 軟性目標）。`docker/runtime/README.md` 記錄版本、建置指令、與上游 `agents-runtime` 的差異。`RuntimeImageIT` 整合測試驗證容器內三個 CLI 的可用性與映像大小。**附帶基礎設施**：新增 `integrationTest` Gradle task（`include "**/*IT.class"`）與 `test` task 的 `exclude("**/*IT.class")` 分離單元測試與整合測試。詳見 `docs/grimo/specs/archive/2026-04-17-S004-grimo-runtime-image.md` 第 7 節。
- **S003 — Sandbox SPI + bind-mount 適配器**（2026-04-17）。採用 `agent-sandbox-core` 0.9.1 的 `Sandbox` SPI 作為沙箱抽象層。實作 `BindMountSandbox`（Testcontainers `GenericContainer` + `withFileSystemBind`），提供 `SandboxManager`（spawn/exec/close）與 `SandboxConfig` record。`BindMountSandboxIT` 整合測試驗證 bind-mount 主機目錄至容器 `/work` 及容器內寫入即時反映於主機。新增 `agent-sandbox-core:0.9.1` 與 `testcontainers:1.20.4` 依賴。詳見 `docs/grimo/specs/archive/2026-04-17-S003-sandbox-bind-mount-adapter.md` 第 7 節。
- **S002 — 模組骨架 + Modulith verify 通過**（2026-04-16）。為 6 個 MVP 模組（`core` OPEN + `sandbox`、`cli`、`agent`、`subagent`、`skills` 皆 `allowedDependencies = {}` 嚴格起步）各建一個 `package-info.java`，建立 Spring Modulith 邊界基線。`ModuleArchitectureTest` 在每次 `./gradlew test` 跑 `ApplicationModules.of(GrimoApplication.class).verify()` 守護政策；`DocumentationTests` 透過 `Documenter` 在 `build/spring-modulith-docs/` 產生 13 個成品（`components.puml` + 每模組 `.adoc` × 6 + 每模組 `.puml` × 6）。**Cross-Module Communication Policy** 於本規格釘定並寫入 `development-standards.md` §13 為權威單一來源（三模式：`core` 直接型別 / `@NamedInterface("api")` 同步埠 / `@NamedInterface("events")` 非同步事件）。Doc-sync 同步將 `architecture.md` §1+§2、`PRD.md` AC10、`glossary.md`（Application Module 擴寫 + Named Interface + Allowed Dependencies 新增）對齊 MVP 6 模組。AC-2（違規偵測）以一次性手動證明完成，捕獲 `org.springframework.modulith.core.Violations: - Module 'subagent' depends on module 'cli' …` 訊息為審計留痕。**附帶修正：** 發現 `native` 是 Java 保留字無法當套件名，將 Backlog 模組改名為 `nativeimage` 並修正 dev-standards.md §12 PR 檢查清單中的舊參照。**無新依賴**（`spring-modulith-starter-core` + `starter-test` 已於 S000 引入；`Documenter` 透過 `spring-modulith-docs` 傳遞引入）。詳見 `docs/grimo/specs/archive/2026-04-16-S002-module-skeleton-modulith-verify.md` 第 7 節。
- **S001 — 核心領域原語 + GrimoHomePaths**（2026-04-16）。新增 `io.github.samzhu.grimo.core.domain` 套件：四個帶型別的 id records（`SessionId`、`TurnId`、`TaskId`、`CorrelationId`）、兩個列舉（`AgentRole { MAIN, SUB, JURY_MEMBER }`、`ProviderId { CLAUDE, CODEX, GEMINI }`）、一個內建的 `NanoIds` 產生器（21 字元 URL 安全 id，約 30 LOC，零新執行期依賴），以及 `GrimoHomePaths` — 一個針對 `~/.grimo/{memory, skills, sessions, worktrees, logs, config, db}` 的靜態解析器，支援 `grimo.home` JVM 系統屬性與 `$GRIMO_HOME` 環境變數覆寫（優先順序：property → env → `$HOME/.grimo`）。ArchUnit 的 `DomainArchitectureTest` 守護 AC-3（`core.domain` 中無 Spring / `jakarta.annotation` import）。所有領域型別均為不可變 records 或純列舉，零 Spring 注解。AC-4（`Cost` 算術）延後至擁有成本遙測的規格（S019）。詳見 `docs/grimo/specs/archive/2026-04-16-S001-core-domain-primitives.md` 第 7 節的關鍵使用模式與測試配方。
- **S000 — 專案初始化**（2026-04-16）。Spring Boot 4.0.5 模組化骨架，運行於 JDK 25，使用 Gradle 9.4.1 wrapper 與 GraalVM native 外掛 0.11.5（`graalvmNative { imageName = "grimo"; buildArgs += "--no-fallback" }`）。透過 `spring.threads.virtual.enabled=true` 啟用 virtual thread 執行器。Hello-world `GrimoApplication` 加上兩個 JUnit 測試（`contextLoads`、`virtualThreadsEnabled`）。`.gitignore` 強化（macOS / `~/.grimo/` 雜訊）。頂層 `README.md` 記錄原生可執行檔打包路徑。AC-3（自動格式化閘門）與 AC-4（實際原生二進位執行）有意延後 — 詳見 `docs/grimo/specs/archive/2026-04-16-S000-project-init.md` 第 7 節。
